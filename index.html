Fokus 1:
Mapbox.com

Online-Service für Karten und Navigation

  • ideal für Web-Projekte (JS SDK: Mapbox GL JS)
  • Die Basics: Kartenmaterial selbst / optimiert für verschiedene Screens / gut dokumentiert
  • Andere Libraries bieten das auch (z.B. Leaflet)

Warum Mapbox?

  • Online Editor zur Gestaltung von Background Maps
  • Einfache Nutzung von Vector Tiles und Raster Tiles
  • Geocoding API
  • Warum also Mapbox?

Mapbox?

  • Neues Lizenzmodell für Mapbox GL JS seit v2
    (nicht mehr komplett Open Source)
  • Trotzdem noch nutzbar durch v1 (BSD-Lizenz) / v2 bis zu 50.000 Map Loads pro Monat (geht für uns klar)
  • Alternativen: MapLibreGL (Fork von Mapbox v1) / Leaflet
  • Für Tilesets braucht man auch Alternative (z.B. eigener Tileserver -> recht aufwendig)
  • Bleiben erstmal bei Mapbox, aber beobachten die Entwicklung
  • Jetzt: Beispiel, warum wir Mapbox einsetzen

From GeoJSON
to Vector Tileset

  • Temperatur-Daten für Berlin für jede Stunde von 10 bis 20 Uhr
  • Man sieht Properties: 9 Uhr -> 21 Uhr / Polygone
  • 28 MB GeoJSON -> Performance-Problem im Browser, Karte nicht nutzbar
  • Mit Mapbox: GeoJSON hochladen und man bekommt automatisch ein Vector Tileset generiert
  • Tileset: Geodaten, die in kleine Quadrate aufteilt werden, die für verschiedene Zoom-Studen optimiert sind
  • Beispiel: man sieht Flächen mit Werten (1-5) für jede Stunde
  • Jedes Tileset hat ID
  • JavaScript-Objekt, das die Informationen für Mapbox GL bereitstellt
  • Tileset URL in der Mitte
  • Dadrunter: Farb-Map für verschiedene Blautöne
  • THF hat leichten Blauton -> mittelmäßig kühl um 17 Uhr
  • Häuserblock daneben hell -> wärmer
